# React Flowmap 소개

## 한 줄 소개

**React Flowmap**는 React 애플리케이션의 실행 흐름을 추적해 컴포넌트, 훅, 함수, API 요청 사이의 관계를 문서화하고 시각화하는 런타임 분석 라이브러리입니다.

> **The Missing Link of Your React Runtime**

---

## React Flowmap이 풀고자 하는 문제

현대적인 프론트엔드 애플리케이션은 수많은 컴포넌트, 커스텀 훅, 유틸 함수, 데이터 요청 로직으로 구성됩니다. 코드가 여러 파일과 계층으로 분산되면서 다음과 같은 문제가 반복됩니다.

- 어떤 화면이 어떤 훅과 함수를 거쳐 어떤 API를 호출하는지 한 번에 보기 어렵습니다.
- 정적 코드 분석만으로는 실제 실행 시점의 관계를 완전히 복원하기 어렵습니다.
- 문서는 빠르게 낡고, 실제 구조와 설계 문서 사이의 간극은 계속 벌어집니다.
- 특정 함수나 API를 수정했을 때 어떤 화면과 기능이 영향을 받는지 추적하기 어렵습니다.

React Flowmap은 이 문제를 "코드를 읽어서 추측하는 방식"이 아니라, **실행 중인 애플리케이션을 관찰하는 방식**으로 풀고자 합니다.

---

## 왜 런타임 분석인가

정적 분석은 유용하지만 한계가 분명합니다. 실제 애플리케이션은 조건 분기, 동적 호출, 커스텀 훅 조합, 상태 변화, 네트워크 요청 같은 런타임 맥락 속에서 동작합니다.

React Flowmap은 애플리케이션이 실행되는 순간 발생하는 연결을 포착합니다.

- 어떤 컴포넌트가 렌더링되었는지
- 어떤 훅이 어떤 흐름 안에서 사용되었는지
- 어떤 함수가 어떤 함수와 API 호출로 이어졌는지
- 어떤 화면이 어떤 네트워크 요청에 의존하는지

즉, React Flowmap은 코드베이스의 구조를 "정의된 형태"가 아니라 **실제로 살아 움직이는 형태**로 드러냅니다.

---

## React Flowmap의 핵심 관점

React Flowmap은 애플리케이션의 관계를 **Node**와 **Edge**로 표현합니다.

### Nodes

- `Component`
- `Hook`
- `Function`
- `API Endpoint`

### Edges

- `Render`
- `Use`
- `Call`
- `Request`

이 모델을 통해 React Flowmap은 단순한 로그 수집 도구가 아니라, 애플리케이션 아키텍처를 탐색 가능한 관계 그래프로 전환합니다.

예를 들어 다음과 같은 흐름을 시각적으로 이해할 수 있습니다.

```text
[UserPage] -> [useUser] -> [fetchUserInfo] -> [GET /api/user]
    |
    +-> [UserForm] -> [updateUser] -> [POST /api/user]
```

---

## React Flowmap이 제공하려는 가치

### 1. 실행 구조의 투명성

애플리케이션 내부의 숨겨진 연결 고리를 드러내어, 대규모 코드베이스의 실제 구조를 빠르게 파악할 수 있도록 돕습니다.

### 2. 항상 최신인 문서

문서를 사람이 수동으로 관리하는 대신, 실행 흐름에서 관계를 수집해 구조 문서를 자동으로 갱신합니다.

### 3. 빠른 영향도 분석

특정 API, 훅, 함수가 변경될 때 어떤 화면과 기능이 영향을 받는지 즉시 파악할 수 있습니다.

### 4. 팀 단위의 공통 이해

개발자, 리뷰어, 신규 팀원이 시스템의 흐름을 동일한 시각 자료로 공유할 수 있습니다.

## React Flowmap의 포지셔닝

React Flowmap은 다음과 같은 질문에 답하는 도구입니다.

- 이 화면은 어떤 데이터 흐름 위에서 동작하는가?
- 이 훅은 어디에서 사용되고 무엇을 호출하는가?
- 이 API 변경은 어떤 컴포넌트에 영향을 주는가?
- 현재 애플리케이션의 실제 실행 구조는 어떻게 생겼는가?

비유하자면, React Flowmap은 프론트엔드 실행 구조를 위한 실시간 명세 도구입니다. API 생태계에서 Swagger가 계약과 구조를 보여주듯, React Flowmap은 애플리케이션 내부 실행 관계를 드러내는 역할을 지향합니다.

---

## 현재 범위와 앞으로의 방향

### 현재 지원

React Flowmap은 **Vite + React 앱**과 **Next.js App Router**를 현재 제품 범위로 둡니다.

| 환경               | 상태                              |
| ------------------ | --------------------------------- |
| Vite + React       | ✅ 지원                           |
| Next.js App Router | ⚠️ `'use client'` 컴포넌트만 추적 |

### 변환 방식

빌드 타임에 **Babel AST 변환**을 사용합니다. 별도의 babel 설정 없이 번들러 플러그인만으로 동작합니다.

- Vite: `flowmapInspect()` 플러그인 (`react-flowmap/vite`)
- Next.js: `withFlowmap()` config 래퍼 (`react-flowmap/next`)

### 향후 방향

- Vite React / Next.js App Router 안정화
- npm 배포
- 일반적인 라우터 환경(TanStack Router, react-router-dom) 검증
- Vue 등 다른 프레임워크 확장은 장기 고려 사항

## React Flowmap이 지향하는 경험

React Flowmap은 개발자가 더 이상 구조를 추측하지 않도록 만드는 것을 목표로 합니다.

- 코드를 따라가며 구조를 머릿속으로 복원하는 대신
- 실행 결과를 통해 관계를 확인하고
- 화면, 훅, 함수, API 사이의 연결을 즉시 이해하며
- 변경의 영향을 더 빠르게 판단할 수 있도록

React Flowmap은 개발 경험의 중심을 "추측"에서 "가시성"으로 옮기려 합니다.

---

## 요약

React Flowmap은 React 애플리케이션의 실행 관계를 추적해, 문서화하고, 시각화하는 런타임 분석 도구입니다. 이 프로젝트의 핵심은 단순한 추적이 아니라, **분산된 코드의 연결 구조를 하나의 명확한 형태로 드러내는 것**입니다.

**Stop guessing. Start seeing with React Flowmap.**
