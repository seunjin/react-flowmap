# React Flowmap 소개

## 한 줄 소개

**React Flowmap**는 현재 화면의 UI 조각이 어떤 컴포넌트에 의해 그려지고 어떤 구조로 조립되어 있는지 보여주는 React 컴포넌트 구조 검사기입니다.

> **See the screen through its component boundaries.**

---

## React Flowmap이 풀고자 하는 문제

프론트엔드 개발자는 화면을 보고도 다음을 바로 답하기 어렵습니다.

- 지금 이 UI 조각은 어느 컴포넌트가 소유하고 있는가
- 현재 화면은 어떤 부모 / 자식 컴포넌트 조합으로 조립되어 있는가
- 이 부분은 이미 별도 컴포넌트로 분리돼 있는가, 아니면 아직 부모 안의 직접 마크업인가
- 이 화면을 더 작은 UI 파츠로 컴포넌트화할 수 있는 지점은 어디인가

코드베이스가 커질수록 이 질문은 더 자주 나오지만, 코드를 위아래로 따라가며 추측하는 방식은 느리고 피로합니다.

React Flowmap은 이 문제를 **화면에서 시작해 구조를 역으로 읽는 방식**으로 풉니다.

---

## 왜 런타임 관찰이 필요한가

React Flowmap의 핵심은 "런타임 분석 플랫폼"이 아니라 **현재 화면 구조를 정확하게 보여주는 것**입니다.

정적 import 트리만으로는 다음을 놓치기 쉽습니다.

- 실제로 지금 화면에 붙어 있는 컴포넌트
- 조건 분기 뒤에서 나타난 컴포넌트
- layout / page 아래에서 현재 활성 라우트 subtree를 구성하는 구조
- 같은 symbol이라도 현재 화면에서 실제로 관찰되는 instance

따라서 React Flowmap은 실행 중인 화면을 관찰해 **현재 보고 있는 UI의 실구조**를 보여줍니다.

---

## React Flowmap의 핵심 가치

### 1. 화면에서 시작하는 구조 이해

코드가 아니라 화면의 한 조각에서 출발해, 그 조각이 어떤 컴포넌트에 속하는지 바로 확인할 수 있습니다.

### 2. 현재 화면 기준의 구조 가시성

앱 전체의 추상 그래프보다, 지금 렌더된 route subtree 안에서 화면이 어떻게 조립되어 있는지 파악하는 데 집중합니다.

### 3. 컴포넌트화 판단 보조

어디까지가 이미 분리된 컴포넌트 영역이고, 어디부터가 아직 부모가 직접 들고 있는 큰 UI 덩어리인지 추론할 수 있도록 돕습니다.

### 4. 즉시 수정 가능한 개발 흐름

props와 TypeScript type metadata를 보고, 바로 소스 파일로 이동해 수정할 수 있습니다.

---

## React Flowmap의 포지셔닝

React Flowmap은 다음 질문에 답하는 도구입니다.

- 지금 내가 집은 UI는 어떤 컴포넌트인가?
- 현재 화면은 어떤 컴포넌트 구조로 조립되어 있는가?
- 현재 라우트는 어떤 layout / page에서 시작되는가?
- 이 구조를 보고 어디를 더 컴포넌트화할 수 있을까?

React Flowmap은 다음을 제품의 1차 목표로 두지 않습니다.

- 함수 호출 그래프 분석
- 네트워크 요청 분석
- 범용 런타임 계측 플랫폼
- Chrome DevTools 대체

---

## 현재 범위와 앞으로의 방향

### 현재 지원

React Flowmap은 **Vite + React 앱**과 **Next.js App Router**를 현재 제품 범위로 둡니다.

| 환경               | 상태                              |
| ------------------ | --------------------------------- |
| Vite + React       | ✅ 지원                           |
| Next.js App Router | ⚠️ `'use client'` 컴포넌트만 추적 |

### 현재 잘해야 하는 것

- 화면에서 UI 조각 pick
- owning component 확인
- live props + TypeScript props type 표시
- 현재 route subtree 기준 graph / explorer 제공
- source jump
- route / layout context 제공

### 후순위 또는 비핵심

- request 목록을 기본 inspector 정보로 노출
- hook / request 숫자 요약
- 범용 함수 / API dependency graph
- DOM 태그 단위 시각화

---

## React Flowmap이 지향하는 경험

React Flowmap은 개발자가 더 이상 구조를 추측하지 않도록 만드는 것을 목표로 합니다.

- 화면에서 한 조각을 집고
- 그 조각의 컴포넌트 정체성과 props를 읽고
- 현재 화면 구조 안에서 어디에 놓여 있는지 보고
- 더 컴포넌트화할 수 있는 지점을 판단하게 만든다

즉, React Flowmap은 **숲보다 나무를 먼저 보여주고, 그 다음 숲을 읽게 하는 도구**입니다.

---

## 요약

React Flowmap은 React 애플리케이션의 현재 화면을 컴포넌트 경계와 구조 관점에서 보여주는 시각적 검사기입니다. 이 프로젝트의 핵심은 더 많은 런타임 신호를 수집하는 것이 아니라, **현재 화면이 어떤 컴포넌트 구조로 서 있는지 명확하게 보이게 만드는 것**입니다.
